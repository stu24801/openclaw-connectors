
## 16:44 DeepWiki Debug Session

### 問題：No valid XML found in response

長時間 debug，找到多個問題層：

1. **Port 8001 crash loop**（`deepwiki-api.service` restart counter > 400）
   - 修法：`/etc/systemd/system/deepwiki-api.service` 加 `ExecStartPre=fuser -k 8001/tcp` + `KillMode=control-group`

2. **LLM Proxy `/v1/responses` 端點缺失**
   - adalflow 1.1.3 改用 OpenAI Responses API (`responses.create`)，但 proxy 只有 `/v1/chat/completions`
   - 修法：`/opt/llm-proxy/main.py` 新增 `/v1/responses` shim，把 Responses API 格式轉成 chat completions 呼叫

3. **Auth code WebSocket 驗證失敗（根本原因）**
   - `LoginPage.tsx` 用 `sessionStorage`（分頁獨立），新分頁 auth code 消失
   - 修法：改成 `localStorage`（三個檔案）

4. **React useCallback closure bug（最終根本原因）**
   - `determineWikiStructure` 和 `generatePageContent` 的 dependency array 沒有包含 `authCode`
   - 導致 closure 永遠捕捉初始值 `''`，WebSocket 送出 `authorization_code=None`
   - 修法：兩個 useCallback 的 dependency array 加入 `authCode`
   - 同時修：主要 data loading useEffect 要等 `isAuthLoading=false` 才執行

## 17:16 Ask 功能錯誤排查

### 問題：右下角提問出現 "Error with Openai API: An error occurred during streaming"

**根本原因**：多層問題疊加
1. **Auth code None** — React useCallback closure bug（authCode 沒加進 dependency array），WebSocket 送出時 authCode = ''
2. **"An error occurred during streaming"** 是 openai SDK 內部訊息，exception catch 後 `str(e)` 只印這段，沒有 traceback，導致難以診斷
3. **LLM Proxy 沒有 systemd 保護** — proxy 是手動 `uvicorn` 跑（PID 306253, `--host 0.0.0.0`），systemd service 是 disabled。proxy 掛了不會自動重啟

**修法（本次完成）**：
- `websocket_wiki.py` except 區塊加 `traceback.format_exc()` 印完整 stack trace
- 確認 ask 功能在 auth 修好後（09:05）正常回應
