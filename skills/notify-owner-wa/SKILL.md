# Notify Owner via WhatsApp（文字通知景揚）

工作完成後，透過 **WhatsApp 文字訊息**主動通知景揚，報告任務結果。

底層使用 `message` 工具（WhatsApp channel），直接發送至景揚的 WhatsApp。

---

## 使用時機

- **工程蝦**：系統修復完成、功能部署上線、設定檔更新完畢
- **寫手蝦**：文章撰寫 / 修改完成、回饋回覆送出、新版文章上傳
- **任何 Agent**：背景任務、Cron Job 或多步驟工作流程執行完畢，需主動通報時

> 若需語音播報，請改用 `job-done-call` skill。

---

## 通知訊息格式

訊息需簡短有力、一眼看懂，格式如下：

```
✅ <任務名稱>已完成

<一句話摘要執行結果>

<若有重要細節或連結，補充一行>
```

**成功範例（工程蝦）：**

```
✅ 回饋功能防連點機制已完成

已在送出按鈕加上狀態鎖定，送出後不會再誤觸。

重整頁面（F5）即可生效。
```

**成功範例（寫手蝦）：**

```
✅ 文章修改完成

已依照回饋調整硬體架構圖，補上 GitHub Copilot 與 SharePoint 連線。

👉 https://rag.alex-stu24801.com/articles/99808286-0335-49ab-9637-5af3ee650dcd
```

**失敗 / 異常範例：**

```
⚠️ 文章更新遇到問題

嘗試儲存時發生權限錯誤，尚未完成更新。

請確認 /opt/rag-kb/data/articles/ 目錄權限後再試。
```

---

## 執行方式

使用 `message` 工具，channel 指定 `whatsapp`，target 指定景揚的 WhatsApp 號碼：

```
message action=send
  channel=whatsapp
  target=<OWNER_WHATSAPP_NUMBER>
  message=<上方格式組合的通知訊息>
```

---

## 語言規範

- **一律使用台灣繁體中文（台灣口語）**
- 語氣自然親切，像台灣人日常傳訊息的感覺
- 訊息長度控制在 **5 行以內**，不需要長篇大論
- 若有相關頁面連結（如文章庫網址），附在最後一行
- 技術術語用白話替代：
  - ✅「已上線」「搞定了」「出了點問題」「幫你弄好了」
  - ❌「已部署完畢」「執行 CI/CD pipeline」

---

## 注意事項

- 不要在凌晨 00:00～07:00 傳送通知（除非景揚明確要求）
- 訊息盡量精簡，不需要重複任務的細節，景揚可以從對話視窗查看完整記錄
- 若 WhatsApp 通道不可用，改用目前對話頻道（如 TUI）直接回覆
